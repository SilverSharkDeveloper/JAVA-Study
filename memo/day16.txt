컬렉션 프레임워크 (Collection Framework)
	많은 데이터를 쉽고 효과적으로 관리할 수 있는 표준화된 방법을 제공하는 클래스들의 집합.

1.List extends Collection
	
	List 인터페이스 구현체

	-Vector : 용량관리, 보안성 강화, 처리량 감소 -> 싱글쓰레드사용 -> 데이터를 저장할때 하나가 모두 저장된 후에 다음 데이터 저장 
	-LinkedList : FILO로 인해 데이터 추가 삭제는 빠름 , 원하는 위치의 데이터를 가져오는것은 상대적으로 느림
	-ArrayList : 인덱스로 데이터를 관리한다. 컬렉션 클래스 중 실무에서 가장 많이 사용되는 클래스이다.
			배열의 특징인 인덱스를 이용하여 값을 저장하고 관리한다.

	배열과 ArrayList의 차이
		배열은 길이에 제한을 두어야할 때 자주 사용되고, ArrayList는 몇 개의 데이터가 들어올 지 알 수 없을 때 사용한다.


제네릭 (이름이없는) -> 어떤 데이터가 들어올지 모르므로 타입을 미리 정해줄 수 없다 -> 먼저 제네릭으로 선언한 후 그 클래스 객체를 사용하는 쪽에서 타입을 제한
	객체화 시 타입을 지정하는 기법 -> 설계할 때에는 타입을 지정할 수 없기 때문에 임시로 타입을 선언할 때 사용한다.

	1.다운캐스팅 안해도댐
	2.<T extends Number> -> Number타입의 타입만 들어오게 제한할 수 있다.
