소프트웨어 디자인 설계 패턴
	MVC 패턴

1.모델 -> 데이터 베이스에있는 테이블 행 한개를 받아올 객체 (컬럼갯수대로 필드를 가진 객체) ->클래스 이름뒤에 VO, DTO가 붙는다.
		VO : valueObject -> 값 그자체를 가지며 불변하고 데이터베이스의 컬럼값과 동일한 데이터만을 가진다.
		DTO : Datat Transfer Object -> 데이터를 뿌려주기위한 객체로 테이블의 조인이나 여러 연산이후 화면에 뿌리고싶은경우 사용한다.

2.뷰(화면) : 사용자에게 보여질 화면을 구성하는 부분	
		-컨트롤러에 선언된 CRUD 메소드를 사용하는 부분
3.컨트롤러 -
	DAO(Data Access Object) 테이블에 접근할 수 있는 메소드들이 선언된 클래스
			접근 후 결과 값이 있을 경우 Model 객체에 담은 후 처리
			클래스명 뒤에 DAO라는 문자열을 붙여준다.
